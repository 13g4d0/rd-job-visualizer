# Session Save

Capturar y persistir todo lo aprendido en esta sesión antes de cerrarla.

## Instrucciones

1. **Revisa la conversación completa** y extrae:
   - Decisiones tomadas (técnicas, de producto, de proceso)
   - Feedback del usuario (correcciones, preferencias, confirmaciones)
   - Nuevos patrones o convenciones descubiertos
   - Áreas frágiles identificadas
   - Cambios en la arquitectura del repo
   - Estado actual del trabajo (qué quedó pendiente, qué se completó)

2. **Actualiza `ORCHESTRATOR.md`** con:
   - Nuevas decisiones en la tabla de decisiones
   - Cambios en la arquitectura si hubo
   - Nuevas áreas frágiles descubiertas
   - Estado actual actualizado (tareas completadas/pendientes)
   - Participantes nuevos o cambios en el equipo
   - Cualquier insight no obvio que un futuro agente necesitaría

3. **Guarda memories persistentes** (en `~/.claude/projects/`) para:
   - Feedback del usuario que aplica a futuras sesiones (tipo `feedback`)
   - Información del usuario relevante (tipo `user`)
   - Decisiones de proyecto no obvias (tipo `project`)
   - Referencias externas descubiertas (tipo `reference`)

4. **Verifica que `BACKLOG.md`** esté al día con el estado real de las tareas.

5. **Verifica que `CLAUDE.md`** refleje el estado actual del proyecto (stack, convenciones, estado).

6. **Muestra un resumen** al usuario de lo que se persistió, organizado por categoría.
